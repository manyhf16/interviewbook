# 乐观锁和悲观锁的理解

# 参考解答

首先看一个著名的 **丢失更新** 问题：
两个用户同时修改一条数据，原数据为：
```
id    name    age
1     张三     20
```

|时间|执行用户|sql|解析|
|-|-|-|-|
|t1|用户1|select * from stu where id=1|用户1看到的数据为1,张三,20|
|t2|用户2|select * from stu where id=1|用户1看到的数据为1,张三,20|
|t3|用户2|update stu set name='张三',age=21 where id=1|以自己看到的数据，执行更新，希望将年龄改为21|
|t4|用户2|commit||
